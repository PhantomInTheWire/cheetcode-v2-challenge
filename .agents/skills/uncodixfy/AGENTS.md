> Guidelines for building non-Codex UI - avoiding AI-generated aesthetic patterns and creating human-designed, functional interfaces.

---

## Overview

This skill provides comprehensive guidelines for breaking away from the default "Codex AI" aesthetic and building interfaces that feel human-designed, functional, and honest. It teaches you to recognize and avoid common AI UI patterns.

## When to Apply

Reference these guidelines when:

- Building new UI components or pages
- Designing frontend interfaces
- Creating dashboards, forms, or navigation
- Styling any web interface
- Reviewing UI code for aesthetic patterns

## Key Principles

1. **Keep It Normal** - Follow standard, functional UI patterns (Linear, Raycast, Stripe, GitHub style)
2. **Hard No** - Avoid all the default AI aesthetic patterns
3. **Specifically Banned** - Don't repeat common mistakes
4. **Color Priority** - Use existing project colors first, then predefined palettes

## Quick Reference

- No oversized rounded corners (max 8-12px)
- No glassmorphism or floating panels
- No eyebrow labels or decorative copy
- No hero sections in dashboards
- No transform animations on hover
- No dramatic shadows
- Use predefined color palettes when needed
